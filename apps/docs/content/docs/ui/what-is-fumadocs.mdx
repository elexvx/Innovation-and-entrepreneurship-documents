Fumadocs 的创建是因为创作者 Fuma 希望有一个更可定制的文档构建体验，成为一个不固执己见的文档框架，**一个你可以"打破"的"框架"**。

## 哲学

### 更少的抽象

虽然大多数框架都通过配置文件进行配置，但当你希望调整其细节时，它们通常缺乏灵活性。
你无法控制它们如何渲染页面也无法控制内部逻辑。

Fumadocs 期望你编写代码并与软件的其他部分协作，它向你展示应用程序的工作原理并且完全可定制，而不是仅仅通过配置文件。

### 无缝集成

Fumadocs 与你的 React 框架紧密集成，带来有用的工具和美观的 UI。

对于 Next.js 开发者，你仍然在使用 App Router 的功能，比如**静态站点生成**。对于 Next.js 开发者来说没有什么新东西，一切都很熟悉。

### 可组合的 UI

Fumadocs UI（默认主题）提供的唯一功能是**用户界面**。UI 是有主见的，旨在带来更好的移动响应性和用户体验。

相反，我们遵循一种更加灵活的方法，受 Shadcn UI 的启发 — [Fumadocs CLI](/docs/cli)，允许你"分叉" Fumadocs UI 的一部分，并完全自定义它。

### 服务器优先的方法

传统上，静态站点生成器是**静态的**。

然而，Fumadocs 引入了由 React Server Component 驱动的服务器优先方法。
通过完美的服务器-客户端边界，内容变得动态和交互式。

你可以从服务器获取数据来显示内容，或与 CMS 集成接收实时更新。它始终是最新的且易于维护。

### 最小化

Fumadocs 由 Fuma 和许多贡献者维护，关注代码库的可维护性。

虽然我们不打算提供人们想要的每一个功能，但我们更专注于让基本功能完美且维护良好。
你也可以通过贡献来帮助 Fumadocs 变得更有用！

## 何时使用 Fumadocs

Fumadocs 的设计考虑了灵活性，它不限于某些用途。

- `fumadocs-core` 是一个用于构建文档的无头 UI 库。
- `fumadocs-mdx` 是一个处理 MDX 内容的有用库。

对于大多数 Web 应用程序，单纯的 React.js 已经不够了。
如今，我们还希望有一个博客、展示页面、FAQ 页面等。在这些情况下，Fumadocs 可以帮助你更轻松地构建文档，减少样板代码。

如果你感兴趣，可以阅读[比较](/docs/ui/comparisons)。

### 文档

Fumadocs 专注于**编写体验**，它提供美观的主题和许多文档自动化工具。

它帮助你更快地迭代代码库，同时永远不会丢下你的文档。
你可以将此站点作为使用 Fumadocs 构建的文档站点的示例。

### 博客站点

大多数 React.js 框架已经可以满足博客站点的需求。

Fumadocs 提供额外的工具，包括语法高亮、文档搜索和默认主题（Fumadocs UI）。
它帮助你避免重新发明轮子。
