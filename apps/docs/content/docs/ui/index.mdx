## 介绍

Fumadocs <span className='text-fd-muted-foreground text-sm'>(Foo-ma docs)</span> 是一个**文档框架**，设计为快速、灵活，
并能无缝集成到你的 React 框架中。

Fumadocs 包含以下几个部分：

<Cards>

<Card icon={<CpuIcon className="text-purple-300" />} title='Fumadocs Core'>

处理大部分逻辑，包括文档搜索、内容源适配器和 Markdown 扩展。

</Card>

<Card icon={<PanelsTopLeft className="text-blue-300" />} title='Fumadocs UI'>

Fumadocs 的默认主题，为文档站点提供美观的外观和交互式组件。

</Card>

<Card icon={<Database />} title='内容源'>

你的内容来源，可以是 CMS 或本地数据层，如 [Fumadocs MDX](/docs/mdx)（官方内容源）。

</Card>

<Card icon={<Terminal />} title='Fumadocs CLI'>

命令行工具，用于安装 UI 组件和自动化操作，对于自定义布局非常有用。

</Card>

</Cards>

<Callout title="想了解更多？">
  阅读我们的深入介绍 [Fumadocs 是什么](/docs/ui/what-is-fumadocs)。
</Callout>

### 术语

**Markdown/MDX：** Markdown 是一种用于创建格式化文本的标记语言。Fumadocs 原生支持 Markdown 和 MDX（Markdown 的超集）。

**[Bun](https://bun.sh)：** 一个 JavaScript 运行时，我们用它来运行脚本。

虽然不是必需的，但具备一些 React.js 的基础知识对进一步的自定义会很有用。

## 自动安装

需要 Node.js 20 或更高版本，注意 Node.js 23.1 可能与 Next.js 存在兼容性问题。

```bash tab="npm"
npm create fumadocs-app
```

```bash tab="pnpm"
pnpm create fumadocs-app
```

```bash tab="yarn"
yarn create fumadocs-app
```

```bash tab="bun"
bun create fumadocs-app
```

它会问你：

- 要使用的 React.js 框架（推荐 Next.js）。
- 要使用的内容源。

一个新的 fumadocs 应用将被初始化。现在你可以开始使用了！

<Callout title='从现有代码库开始？'>

    你可以按照 [手动安装](/docs/ui/manual-installation) 指南来开始。

</Callout>

### 开始使用！

在 docs 文件夹中创建你的第一个 MDX 文件。

```mdx title="content/docs/index.mdx"
---
title: Hello World
---

## Yo what's up
```

在开发模式下运行应用，然后访问 http://localhost:3000/docs。

```npm
npm run dev
```

## 常见问题

一些你可能遇到的常见问题。

<Accordions>

  <Accordion id='change-base-url' title="如何更改文档的基本路由？">

    路由由你的 React 框架处理，你需要先更改路由结构。

    例如，在 Next.js 中，重命名路由（`/docs/*` -> `/info/*`）：

    <Files>
      <Folder name="app/docs" defaultOpen className="opacity-50" disabled>
        <File name="layout.tsx" />
      </Folder>
      <Folder name="app/info" defaultOpen>
        <File name="layout.tsx" />
      </Folder>
    </Files>

    或者使用路由组从 `/docs/*` 重命名为 `/*`：

    <Files>
      <Folder name="app/(docs)" defaultOpen>
        <File name="layout.tsx" />
      </Folder>
    </Files>

    最后，在 `source.ts` 中更新页面的基本 URL：

```ts title="lib/source.ts"
import { loader } from 'fumadocs-core/source';

export const source = loader({
  baseUrl: '/info', // 更改为新值 [!code highlight]
});
```

  </Accordion>

    <Accordion id='multi-docs' title="如何实现多文档？">
        我们推荐使用 [侧边栏选项卡](/docs/ui/navigation/sidebar#sidebar-tabs)。
    </Accordion>

</Accordions>

### 针对 Next.js

<Accordions>
  <Accordion id='dynamic-route' title="它使用动态路由，性能会很差吗？">

    当配置了 `generateStaticParams` 时，Next.js 会将动态路由转换为静态路由。
    因此，它和静态页面一样快。

    你可以在 Next.js 上 [启用静态导出](/docs/ui/static-export) 来获得静态构建输出。

  </Accordion>
  <Accordion id='custom-layout-docs-page' title='如何在 /docs 下创建不使用文档布局的页面？'>

    与在 Next.js App Router 中管理布局相同，从内容目录（`/content/docs`）中删除原始的 MDX 文件。
    这确保了重复的页面不会导致错误。

    现在，你可以将页面添加到另一个路由组中，该组不是文档布局的子级。

    例如，要替换 `/docs/test`：

    <Files>
      <File name="(home)/docs/test/page.tsx" />
      <Folder name="docs">
        <File name="layout.tsx" />
        <File name="[[...slug]]/page.tsx" />
      </Folder>
    </Files>

    对于 `/docs`，你需要将 catch-all 路由更改为非可选的：

    <Files>
      <File name="(home)/docs/page.tsx" />
      <Folder name="docs" defaultOpen>
        <File name="layout.tsx" />
        <File name="[...slug]/page.tsx" />
      </Folder>
    </Files>

  </Accordion>

</Accordions>

### 升级 Fumadocs

当你遇到任何问题或尝试新功能时，请确保升级 Fumadocs：

```bash title="pnpm"
pnpm update -i -r --latest
```
